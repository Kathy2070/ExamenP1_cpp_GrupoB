Metodología:
Fase 1. Análisis: Leer el caso, identificar actores, variables, validaciones, restricciones, reportes y casos críticos. Elaborar una prueba de escritorio mínima de un escenario relevante antes de programar.
Fase 2. Diseño: Definir clases, atributos, métodos, estructuras de almacenamiento, menús, submenús y relación entre el caso principal y el módulo de Ahorcado.
Fase 3. Construcción incremental: Implementar primero la estructura base del proyecto, luego registros y validaciones, después reportes, recursividad, ordenamiento manual y finalmente el módulo Ahorcado.
Fase 4. Pruebas y depuración: Ejecutar casos normales, casos límite y el fragmento defectuoso asignado. Registrar al menos una corrección importante y su explicación.
Fase 5. Versionamiento: Crear el repositorio local desde GitHub Desktop en Windows, realizar commits significativos durante el desarrollo y publicar el proyecto en GitHub.
Fase 6. Cierre: Entregar el código funcional, la evidencia del repositorio, la prueba de escritorio, la explicación metodológica y defender el programa ante el docente.


Estructura:
•	Registrar vehículos con placa, tipo, capacidad de tanque, kilometraje inicial y estado.
•	Registrar viajes con ruta, distancia, litros abastecidos, litros consumidos, carga y chofer.
•	Detectar inconsistencias, calcular rendimiento km/litro y clasificarlo por nivel.
•	Buscar por placa y por chofer, ordenar manualmente por menor rendimiento promedio y generar alertas de revisión.
•	Emitir reportes de costo operativo, choferes auditados y viajes críticos.
Integración obligatoria con Ahorcado: Construir un Ahorcado con palabras relacionadas con rutas, partes del vehículo, combustible y mantenimiento.
